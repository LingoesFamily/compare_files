# 📁 文件比较器工具

[![Python Version](https://img.shields.io/badge/python-3.7%2B-blue)](https://www.python.org/downloads/)
[![License](https://img.shields.io/badge/license-MIT-green)](LICENSE)
[![Platform](https://img.shields.io/badge/platform-Windows%20%7C%20macOS%20%7C%20Linux-lightgrey)]()

一个简单易用的跨平台桌面工具，用于**比较两个文件夹的差异**，并自动将**新增文件**（仅在基准文件夹中存在的文件）拷贝到指定输出目录，同时生成详细的 CSV 清单。

> 💡 **典型场景**：当你需要同步两个文件夹的内容，或者只想提取某个文件夹相对于另一个文件夹的新增/修改文件时，本工具可以一键完成比较、预览和拷贝。

## ✨ 功能特性

- 🔍 **两种比较模式**
  - 指定基准 vs 指定对比：比较两个完全独立的文件夹。
  - 基准文件夹 vs 对比文件夹（同父目录）：比较同一父目录下的两个子文件夹（如 `a/` 和 `b/`）。

- 🧾 **预览模式**  
  执行前可快速查看统计信息（文件总数、差异数量）以及具体的**增量文件清单**（最多显示前30个），避免误操作。

- 📂 **文件类型过滤**  
  支持按预设类型（图片、视频、文档）过滤，也支持自定义扩展名（如 `jpg,png,mp4`）。同时可排除隐藏文件/目录。

- 🔄 **保留相对路径**  
  拷贝时自动保持源目录的文件夹结构，方便直接覆盖或合并。

- 📄 **生成 CSV 清单**  
  每次比较都会生成带时间戳的 CSV 文件，记录所有文件的状态（新增、冗余、共同）。

- ⏹️ **可随时停止**  
  扫描和拷贝过程中可安全中断，不会损坏已拷贝的文件。

- 🌍 **跨平台支持**  
  基于 Python + Tkinter，在 Windows、macOS、Linux 上均可运行。

## 🚀 快速开始

### 环境要求
- Python 3.7 或更高版本
- 无需额外安装第三方库（仅使用标准库 `tkinter`, `os`, `shutil`, `threading`, `subprocess` 等）

### 下载与运行

1. 克隆仓库或直接下载 `file_comparator.py`：
   ```bash
   git clone https://github.com/yourusername/file-comparator.git
   cd file-comparator
运行程序：

bash
python file_comparator.py
提示：在 macOS/Linux 上如果 tkinter 未预装，请执行 sudo apt-get install python3-tk (Ubuntu/Debian) 或 brew install python-tk (macOS)。

🖥️ 使用指南
1. 选择比较模式
指定基准 vs 指定对比：需要分别选择两个文件夹。

基准文件夹 vs 对比文件夹（同父目录）：只需选择基准文件夹的父目录，然后从下拉框中选择两个子文件夹名称（示例中预置了 a,b,c,d，您可以在代码中修改或直接输入）。

2. 设置路径
基准文件夹：参考文件夹，增量文件将从这里拷贝。

对比文件夹：根据模式选择另一个文件夹或子文件夹。

输出目录：存放生成的 CSV 清单和拷贝的增量文件。

3. 配置选项（可选）
文件类型：可过滤只比较特定类型的文件。

自定义扩展名：例如输入 txt,log,ini。

排除隐藏文件：自动跳过以 . 开头的文件和目录。

保留相对路径：推荐勾选，拷贝时保留源目录结构。

仅生成清单：勾选后只输出 CSV 文件，不实际拷贝文件。

4. 执行操作
点击 预览 → 查看差异统计和增量文件列表。

点击 开始 → 执行完整比较，并自动拷贝增量文件。

点击 停止 → 安全中断当前任务。

打开输出目录 → 在系统文件管理器中打开输出文件夹（支持 Windows/macOS/Linux）。

另存清单 → 将最新的 CSV 清单另存到其他位置。

❓ 常见问题
Q：点击“开始”后没有任何反应？
A：请检查基准文件夹、对比文件夹和输出目录是否都已正确选择，且路径存在。如果问题依旧，请在 GitHub Issues 中反馈。

Q：预览时显示的增量文件数量很多，但我不想拷贝所有文件怎么办？
A：可以勾选“仅生成清单”，先生成 CSV 清单，手动编辑后根据需要拷贝部分文件。或者调整文件类型过滤条件。

Q：打开输出目录失败怎么办？
A：工具会尝试调用系统默认文件管理器。如果失败，会弹出提示框显示目录路径，您可手动在资源管理器中打开该路径。

Q：是否支持比较子文件夹中的文件？
A：支持。程序会递归遍历所有子目录，并根据“保留相对路径”选项拷贝完整的目录结构。

Q：能否在命令行中静默运行？
A：目前仅提供图形界面，暂不支持命令行参数。如需自动化，可基于核心扫描逻辑编写脚本。
